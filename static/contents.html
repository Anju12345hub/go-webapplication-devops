Learn DevOps from Basics

DevOps is a set of practices that combines software development (Dev) and IT operations (Ops)

It aims to shorten the systems development life cycle and provide continuous delivery with high software quality. DevOps is complementary with Agile software development; several DevOps aspects came from Agile methodology

Tools used

Docker

What is Docker?

Docker is a containerization platform that allows you to package applications and their dependencies into lightweight, portable containers.

Why use Docker?

Ensures consistency across development, testing, and production

Reduces setup time by packaging everything into one unit

Improves scalability by running multiple containers efficiently

Works with Kubernetes for easy deployment and orchestration

Github Actions

What is GitHub Actions?

GitHub Actions is a CI/CD (Continuous Integration and Continuous Deployment) tool built into GitHub. It allows you to automate workflows like testing, building, and deploying code when changes are pushed to a repository.

Why use GitHub Actions?

Automates development tasks (e.g., testing, deployment)

Runs directly in GitHub (no need for external CI/CD tools,making it easy to track and debug.)

Supports custom scripts and integrations

Works with Docker, Kubernetes, and cloud providers

Kubernetes

What is Kubernetes?

Kubernetes (K8s) is an open-source container orchestration platform that helps you deploy, manage, and scale containerized applications automatically.

Why use Kubernetes?

You deploy applications as containers in Kubernetes.

Kubernetes manages these containers, ensuring they run properly.

It automatically handles scaling, load balancing, and self-healing (restarting failed containers).

You define everything in YAML files, making deployments predictable and repeatable.

Argo CD

What is ArgoCD?

ArgoCD is a GitOps tool for Kubernetes. It automates application deployment and management using Git as the source of truth.

ArgoCD continuously monitors the Git repository for changes.

If a change is detected, ArgoCD automatically applies it to the Kubernetes cluster.

It ensures that the actual state of the cluster matches the desired state in Git.

Gives a web UI to track and manage applications

Helm charts

What is a Helm Chart?

A Helm Chart is a package for deploying applications on Kubernetes. It contains pre-configured YAML templates that make it easy to install, upgrade, and manage applications.

Why use Helm Charts?

Simplifies Kubernetes deployments with reusable templates

Manages complex applications with a single command

Supports versioning and rollbacks

Allows customization through values files (values.yaml)
